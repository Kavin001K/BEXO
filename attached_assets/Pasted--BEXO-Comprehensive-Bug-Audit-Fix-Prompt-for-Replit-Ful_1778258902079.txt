# SECTION 1 — BLOCKING BUG: GOOGLE AUTH MODULE NOT INSTALLED

## Error
```
Unable to resolve module expo-auth-session/providers/google
from app/(auth)/index.tsx
```

## Root Cause
`expo-auth-session` is imported but **not installed** in node_modules. The package was specified in the Phase 2 build prompt but was never actually installed in the project.

## Fix — Run these commands in the terminal:

```bash
npx expo install expo-auth-session expo-web-browser expo-crypto
```

Then in `app/(auth)/index.tsx`, verify the import is exactly this — no path variations:

```typescript
import * as WebBrowser from 'expo-web-browser';
import * as Google from 'expo-auth-session/providers/google';

WebBrowser.maybeCompleteAuthSession();
```

Then inside the component, wire it properly:

```typescript
const [request, response, promptAsync] = Google.useAuthRequest({
  iosClientId: process.env.EXPO_PUBLIC_GOOGLE_IOS_CLIENT_ID,
  androidClientId: process.env.EXPO_PUBLIC_GOOGLE_ANDROID_CLIENT_ID,
  webClientId: process.env.EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID,
});

useEffect(() => {
  if (response?.type === 'success') {
    const { id_token } = response.params;
    supabase.auth.signInWithIdToken({
      provider: 'google',
      token: id_token,
    });
  }
}, [response]);
```

Add to `.env.local`:
```bash
EXPO_PUBLIC_GOOGLE_IOS_CLIENT_ID=your_ios_client_id_from_google_cloud_console
EXPO_PUBLIC_GOOGLE_ANDROID_CLIENT_ID=your_android_client_id_from_google_cloud_console
EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID=your_web_client_id_from_google_cloud_console
```

3. For iOS: Bundle ID must match `app.json` `ios.bundleIdentifier`
4. For Android: Package name must match `app.json` `android.package`
5. In Supabase Dashboard → Authentication → Providers → Google:
   - Paste your Web Client ID and Web Client Secret
   - Redirect URL from Supabase must be added to Google Console's "Authorized redirect URIs"

# SECTION 2 — CRITICAL: DEPRECATED STYLE PROPS

## Error
```
"shadow*" style props are deprecated. Use "boxShadow".
```

## Root Cause
Multiple components use the old React Native shadow API which is deprecated for web targets.

## Fix — Search the entire codebase for these patterns and replace:

**Find and replace ALL instances of:**
```typescript
// OLD — deprecated
shadowColor: '#000',
shadowOffset: { width: 0, height: 8 },
shadowOpacity: 0.3,
shadowRadius: 16,
elevation: 8,
```

**Replace with the unified cross-platform helper:**

Create `src/utils/shadow.ts`:
```typescript
import { Platform, StyleSheet } from 'react-native';

export const createShadow = (
  color: string = '#000',
  offsetY: number = 8,
  blur: number = 16,
  opacity: number = 0.3,
  elevation: number = 8
) => {
  if (Platform.OS === 'web') {
    return {
      boxShadow: `0 ${offsetY}px ${blur}px ${colorWithOpacity(color, opacity)}`,
    };
  }
  return {
    shadowColor: color,
    shadowOffset: { width: 0, height: offsetY },
    shadowOpacity: opacity,
    shadowRadius: blur / 2,
    elevation,
  };
};

const colorWithOpacity = (hex: string, opacity: number): string => {
  const r = parseInt(hex.slice(1, 3), 16);
  const g = parseInt(hex.slice(3, 5), 16);
  const b = parseInt(hex.slice(5, 7), 16);
  return `rgba(${r},${g},${b},${opacity})`;
};

// Pre-built shadows for common use cases
export const shadows = {
  card: createShadow('#000', 8, 24, 0.25, 6),
  button: createShadow('#7C6AFA', 8, 32, 0.35, 8),
  modal: createShadow('#000', 16, 48, 0.5, 16),
  subtle: createShadow('#000', 2, 8, 0.15, 2),
};
```

Then replace all shadow props in every component file with:
```typescript
import { shadows } from '../../src/utils/shadow';

// In StyleSheet:
card: {
  ...shadows.card,
  // rest of styles
}
```

# SECTION 4 — LOGICAL BUGS (App breaks silently)

## Bug 4.1 — Auth user creation race condition

**Problem:** When a new Google OAuth user signs in, the `onAuthStateChange` listener fires before the user's record can be created in `public.users`. The app tries to route them to `/handle` but there's no guard to prevent multiple rapid fires.

**Fix** — Add debounce and idempotency check in `app/_layout.tsx`:

```typescript
// Add at the top of the auth state change handler:
let isHandlingAuth = false;

const { data: { subscription } } = supabase.auth.onAuthStateChange(
  async (event, session) => {
    if (isHandlingAuth) return; // prevent re-entry
    isHandlingAuth = true;

    try {
      setSession(session);
      setUser(session?.user ?? null);

      if (session?.user) {
        const { data: userRecord } = await supabase
          .from('users')
          .select('id, handle, plan, onboarding_completed, push_token')
          .eq('id', session.user.id)
          .maybeSingle(); // use maybeSingle() not single() — won't throw on no record

        if (userRecord) {
          setUserRecord(userRecord);
          if (userRecord.onboarding_completed) {
            router.replace('/(main)');
          } else {
            router.replace('/(onboarding)/resume');
          }
        } else {
          router.replace('/(auth)/handle');
        }
      } else {
        setUserRecord(null);
        router.replace('/(auth)');
      }
    } finally {
      isHandlingAuth = false;
      setLoading(false);
    }
  }
);
```

## Bug 4.2 — Handle creation doesn't create a profile row

**Problem:** `app/(auth)/handle.tsx` inserts into `public.users` but does NOT create the corresponding `public.profiles` row. Every downstream query (`profiles.avatar_url`, `profiles.theme`, etc.) will return null and potentially crash the dashboard.

**Fix** — In the handle claim handler, always create the profile atomically:

```typescript
const handleClaimHandle = async () => {
  setIsLoading(true);
  const { data: { user } } = await supabase.auth.getUser();
  if (!user) return;

  // Use a transaction-like approach: insert both or fail
  const { error: userError } = await supabase
    .from('users')
    .insert({
      id: user.id,
      email: user.email!,
      handle: handle.toLowerCase().trim(),
      plan: 'free',
      onboarding_completed: false,
    });

  if (userError) {
    // Check if it's a uniqueness violation
    if (userError.code === '23505') {
      setError('That handle is already taken. Try another.');
    } else {
      setError('Something went wrong. Please try again.');
    }
    setIsLoading(false);
    return;
  }

  // CRITICAL: create the profiles row too
  const { error: profileError } = await supabase
    .from('profiles')
    .insert({
      user_id: user.id,
      theme: 'minimal-dark',
      accent_color: '#7C6AFA',
      visibility: 'public',
    });

  if (profileError) {
    console.error('Profile creation failed:', profileError);
    // Non-fatal: profile can be created later. Continue.
  }

  setIsLoading(false);
  router.replace('/(onboarding)/resume');
};
```

## Bug 4.3 — Onboarding store persists stale data after logout

**Problem:** If a user logs out and a new user logs in on the same device, the Zustand `onboardingStore` (persisted to AsyncStorage) still has the previous user's data. The new user would see someone else's resume data pre-filled.

**Fix** — Clear the onboarding store on logout. In `src/store/authStore.ts`, update the `clear` function:

```typescript
import { useOnboardingStore } from './onboardingStore';
import { useBuildStore } from './buildStore';

// Update clear():
clear: () => {
  set({ session: null, user: null, userRecord: null });
  useOnboardingStore.getState().reset();
  useBuildStore.getState().reset();
  // Also clear persisted AsyncStorage keys
  AsyncStorage.multiRemove(['bexo-auth', 'bexo-onboarding']).catch(console.error);
},
```

## Bug 4.4 — OTP verification screen has no error recovery

**Problem:** If `supabase.auth.verifyOtp()` fails, the current code clears the OTP and shows an error. But if the user has already used their one valid OTP (expired or used), there's no way to go back and re-enter their email.

**Fix** — Add a "Use a different email" button that appears after 2 failed attempts:

```typescript
const [failCount, setFailCount] = useState(0);

const handleOtpComplete = async (otp: string) => {
  setIsVerifying(true);

  const { error } = await supabase.auth.verifyOtp({
    email: params.email,
    token: otp,
    type: 'email',
  });

  setIsVerifying(false);

  if (error) {
    triggerShakeAnimation();
    clearOtp();
    setFailCount(prev => prev + 1);

    if (failCount >= 1) {
      setError('Code expired or invalid. Request a new one or use a different email.');
      setShowAlternativeOptions(true); // renders "← Different email" and "Resend" buttons
    } else {
      setError('Invalid code. Please try again.');
    }
  }
};
```

## Bug 4.5 — Resume upload to edge function exposes anon key in error logs

**Problem:** The `resumeParser.ts` service passes the `EXPO_PUBLIC_SUPABASE_ANON_KEY` directly in the Authorization header. While this is technically correct, the full PDF base64 string is also being sent in a single POST. For large PDFs (>5MB), this will timeout at most edge function providers (50MB limit but 10s timeout).

**Fix** — Upload the PDF to Supabase Storage first, then pass the storage path to the edge function:

```typescript
// In resumeParser.ts, change the approach:
export const parseResume = async (
  storagePath: string, // pass the storage path, not local URI
  userId: string,
): Promise<ParsedResume | null> => {
  const response = await fetch(
    `${process.env.EXPO_PUBLIC_SUPABASE_URL}/functions/v1/parse-resume`,
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY}`,
      },
      body: JSON.stringify({
        storage_path: storagePath, // edge function fetches from storage directly
        user_id: userId,
        provider: 'anthropic', // use anthropic as primary (better PDF support)
      }),
    }
  );
  // ...
};
```

## Optimization 7.2 — Debounce handle availability check

The current handle check likely fires on every keystroke — that's a Supabase query on every character.

**Fix:**
```typescript
import { useCallback, useEffect, useState } from 'react';

// Debounce helper
function useDebounce<T>(value: T, delay: number): T {
  const [debouncedValue, setDebouncedValue] = useState<T>(value);
  useEffect(() => {
    const handler = setTimeout(() => setDebouncedValue(value), delay);
    return () => clearTimeout(handler);
  }, [value, delay]);
  return debouncedValue;
}

// In handle screen:
const debouncedHandle = useDebounce(handle, 500);

useEffect(() => {
  if (debouncedHandle.length >= 3) {
    checkHandleAvailability(debouncedHandle);
  }
}, [debouncedHandle]);
```

# SECTION 9 — SUPABASE EDGE FUNCTION: parse-resume

Deploy the resume parser edge function. Create the file at `supabase/functions/parse-resume/index.ts`:

```typescript
import { serve } from 'https://deno.land/std@0.208.0/http/server.ts';
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2';

const corsHeaders = {
  'Access-Control-Allow-Origin': '*',
  'Access-Control-Allow-Headers': 'authorization, x-client-info, apikey, content-type',
};

const EXTRACTION_PROMPT = `You are a precise resume data extractor. Extract structured data from the provided resume PDF.
Return ONLY a valid JSON object — no markdown, no code fences, no explanation.
Omit any field you cannot find (do not return null).

Extract:
{
  "fullName": "string",
  "roles": ["string - 1 or 2 job titles"],
  "bio": "string - 2-3 sentence professional summary",
  "location": "string - City, Country",
  "university": "string",
  "degree": "string - e.g. B.Tech",
  "field": "string - e.g. Computer Science",
  "gradYear": "string - 4-digit year",
  "socialGithub": "string - username only",
  "socialLinkedin": "string - slug only",
  "experiences": [{"company":"","role":"","startDate":"YYYY-MM","endDate":"YYYY-MM or null","isCurrent":false,"description":""}],
  "projects": [{"title":"","description":"","techStack":[],"githubUrl":"","liveUrl":""}],
  "skills": [{"name":"","category":"language|framework|tool|soft|cloud|custom"}]
}

Rules: max 4 experiences, 4 projects, 20 skills. Never invent information.`;

serve(async (req) => {
  if (req.method === 'OPTIONS') {
    return new Response('ok', { headers: corsHeaders });
  }

  try {
    const authHeader = req.headers.get('Authorization');
    if (!authHeader) {
      return new Response(JSON.stringify({ error: 'Unauthorized' }), {
        status: 401, headers: { ...corsHeaders, 'Content-Type': 'application/json' },
      });
    }

    const { storage_path, user_id, provider = 'anthropic' } = await req.json();

    if (!storage_path || !user_id) {
      return new Response(JSON.stringify({ error: 'Missing storage_path or user_id' }), {
        status: 400, headers: { ...corsHeaders, 'Content-Type': 'application/json' },
      });
    }

    // Fetch PDF from Supabase Storage using service role
    const supabase = createClient(
      Deno.env.get('SUPABASE_URL')!,
      Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
    );

    const { data: fileData, error: downloadError } = await supabase.storage
      .from('resumes')
      .download(storage_path);

    if (downloadError || !fileData) {
      throw new Error(`Failed to download resume: ${downloadError?.message}`);
    }

    const arrayBuffer = await fileData.arrayBuffer();
    const base64Pdf = btoa(String.fromCharCode(...new Uint8Array(arrayBuffer)));

    let result: string;

    if (provider === 'openai') {
      const openaiKey = Deno.env.get('OPENAI_API_KEY')!;
      const response = await fetch('https://api.openai.com/v1/chat/completions', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json', 'Authorization': `Bearer ${openaiKey}` },
        body: JSON.stringify({
          model: 'gpt-4o',
          max_tokens: 2000,
          messages: [
            { role: 'system', content: EXTRACTION_PROMPT },
            {
              role: 'user',
              content: [
                { type: 'text', text: 'Extract all data from this resume PDF.' },
                { type: 'image_url', image_url: { url: `data:application/pdf;base64,${base64Pdf}`, detail: 'high' } },
              ],
            },
          ],
        }),
      });
      const data = await response.json();
      result = data.choices?.[0]?.message?.content;
      if (!result) throw new Error('OpenAI returned empty response');
    } else {
      // Anthropic Claude (primary)
      const anthropicKey = Deno.env.get('ANTHROPIC_API_KEY')!;
      const response = await fetch('https://api.anthropic.com/v1/messages', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'x-api-key': anthropicKey,
          'anthropic-version': '2023-06-01',
        },
        body: JSON.stringify({
          model: 'claude-sonnet-4-6',
          max_tokens: 2000,
          system: EXTRACTION_PROMPT,
          messages: [{
            role: 'user',
            content: [
              {
                type: 'document',
                source: { type: 'base64', media_type: 'application/pdf', data: base64Pdf },
              },
              { type: 'text', text: 'Extract all resume data from this document.' },
            ],
          }],
        }),
      });
      const data = await response.json();
      result = data.content?.[0]?.text;
      if (!result) throw new Error('Anthropic returned empty response');
    }

    // Validate parseable JSON
    const parsed = JSON.parse(result);

    return new Response(JSON.stringify({ result: JSON.stringify(parsed) }), {
      headers: { ...corsHeaders, 'Content-Type': 'application/json' },
    });

  } catch (err) {
    console.error('parse-resume error:', err);
    return new Response(JSON.stringify({ error: err.message }), {
      status: 500, headers: { ...corsHeaders, 'Content-Type': 'application/json' },
    });
  }
});
```
