CONTEXT: WHAT WAS BUILT IN PHASE 1
The BEXO app was designed in Phase 1 using React Native + Expo (managed workflow) with:

Expo Router (file-based navigation)
NativeWind + StyleSheet for styling
Moti + React Native Reanimated 3 for animations
Lucide React Native for icons
Static/mock data everywhere — no real API calls

Phase 2 wires every screen to a real backend. The design must NOT change. Colors, fonts, animations, component styles — all stay exactly as built in Phase 1. Only data sources change: from static mock objects to real Supabase queries, real AI calls, and real file uploads.

COMPLETE TECH STACK FOR PHASE 2
Mobile Framework:     Expo SDK 51+ (managed workflow, React Native)
Navigation:           Expo Router v3 (file-based)
Backend/Database:     Supabase (Postgres + Auth + Storage + Edge Functions + Realtime)
Auth:                 Supabase Auth — email OTP + Google OAuth via @supabase/supabase-js
File Storage:         Supabase Storage (avatars bucket, projects bucket, resumes bucket)
AI Resume Parsing:    OpenAI API gpt-4o with vision/document input (PDF → structured JSON)
AI Fallback:          If OpenAI unavailable, use Anthropic Claude claude-sonnet-4-5 API
Portfolio Generation: HTTP webhook to n8n (self-hosted or n8n.cloud)
Push Notifications:   Expo Notifications + Expo Push Token stored in Supabase
Image Compression:    expo-image-manipulator (compress before upload)
PDF Handling:         expo-document-picker + expo-file-system
State Management:     Zustand (lightweight, no Redux complexity)
API Client:           @supabase/supabase-js v2
Environment:          react-native-dotenv + Expo Constants (EAS secrets in production)
Error Tracking:       Sentry (expo-sentry)
Analytics:            PostHog React Native SDK
HTTP Client:          Native fetch (no axios needed with Supabase client)

RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

  BEFORE UPDATE ON public.profiles
  FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();

CREATE TRIGGER update_users_updated_at
  BEFORE UPDATE ON public.users
  FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();

typescriptimport { createClient } from '@supabase/supabase-js';
import AsyncStorage from '@react-native-async-storage/async-storage';
import { Database } from '../types/database.types';
import Constants from 'expo-constants';

const supabaseUrl = Constants.expoConfig?.extra?.supabaseUrl as string;
const supabaseAnonKey = Constants.expoConfig?.extra?.supabaseAnonKey as string;

if (!supabaseUrl || !supabaseAnonKey) {
  throw new Error('Missing Supabase environment variables. Check .env.local');
}

export const supabase = createClient<Database>(supabaseUrl, supabaseAnonKey, {
  auth: {
    storage: AsyncStorage,
    autoRefreshToken: true,
    persistSession: true,
    detectSessionInUrl: false,
  },
  realtime: {
    params: {
      eventsPerSecond: 10,
    },
  },
});

// Helper: get current user ID, throws if not authenticated
export const getCurrentUserId = async (): Promise<string> => {
  const { data: { user }, error } = await supabase.auth.getUser();
  if (error || !user) throw new Error('Not authenticated');
  return user.id;
};

// Helper: get public URL for a storage file
export const getPublicUrl = (bucket: string, path: string): string => {
  const { data } = supabase.storage.from(bucket).getPublicUrl(path);
  return data.publicUrl;
};

4A — Auth Store (src/store/authStore.ts)
typescriptimport { create } from 'zustand';
import { persist, createJSONStorage } from 'zustand/middleware';
import AsyncStorage from '@react-native-async-storage/async-storage';
import { Session, User } from '@supabase/supabase-js';

interface AuthState {
  session: Session | null;
  user: User | null;
  userRecord: {
    id: string;
    handle: string;
    plan: string;
    onboarding_completed: boolean;
    push_token: string | null;
  } | null;
  isLoading: boolean;
  setSession: (session: Session | null) => void;
  setUser: (user: User | null) => void;
  setUserRecord: (record: AuthState['userRecord']) => void;
  setLoading: (loading: boolean) => void;
  clear: () => void;
}

export const useAuthStore = create<AuthState>()(
  persist(
    (set) => ({
      session: null,
      user: null,
      userRecord: null,
      isLoading: true,
      setSession: (session) => set({ session }),
      setUser: (user) => set({ user }),
      setUserRecord: (userRecord) => set({ userRecord }),
      setLoading: (isLoading) => set({ isLoading }),
      clear: () => set({ session: null, user: null, userRecord: null }),
    }),
    {
      name: 'bexo-auth',
      storage: createJSONStorage(() => AsyncStorage),
      partialize: (state) => ({
        userRecord: state.userRecord,
      }),
    }
  )
);

interface UploadResult {
  url: string;
  path: string;
}

/**
 * Compress an image and upload to Supabase Storage.
 * Shows compression progress in the UI via the onProgress callback.
 */
export const uploadImage = async (
  localUri: string,
  bucket: 'avatars' | 'projects',
  userId: string,
  filename: string,
  onProgress?: (progress: number) => void,
): Promise<UploadResult> => {
  // Step 1: Compress the image
  onProgress?.(10);
  const compressed = await ImageManipulator.manipulateAsync(
    localUri,
    [{ resize: { width: bucket === 'avatars' ? 400 : 1200 } }],
    {
      compress: 0.82,
      format: ImageManipulator.SaveFormat.JPEG,
    }
  );

  onProgress?.(40);

  // Step 2: Read as base64
  const base64 = await FileSystem.readAsStringAsync(compressed.uri, {
    encoding: FileSystem.EncodingType.Base64,
  });

  onProgress?.(60);

  // Step 3: Upload to Supabase Storage
  const filePath = `${userId}/${filename}-${Date.now()}.jpg`;
  const { error } = await supabase.storage
    .from(bucket)
    .upload(filePath, decode(base64), {
      contentType: 'image/jpeg',
      upsert: true,
    });

  if (error) throw new Error(`Upload failed: ${error.message}`);

  onProgress?.(90);

  // Step 4: Get public URL
  const { data } = supabase.storage.from(bucket).getPublicUrl(filePath);

  onProgress?.(100);

  return { url: data.publicUrl, path: filePath };
};

/**

 * Falls back to Anthropic Claude if OpenAI fails.

export const parseResume = async (localPdfUri: string): Promise<ParsedResume | null> => {
  try {
    // Read PDF as base64
    const base64Pdf = await FileSystem.readAsStringAsync(localPdfUri, {
      encoding: FileSystem.EncodingType.Base64,
    });

    // Try OpenAI first
    try {
      const result = await parseWithOpenAI(base64Pdf);
      if (result) return result;
    } catch (openAiError) {
      console.warn('OpenAI resume parse failed, trying Anthropic:', openAiError);
    }

    // Fallback to Anthropic
    const result = await parseWithAnthropic(base64Pdf);
    return result;

  } catch (err) {
    console.error('Resume parsing failed entirely:', err);
    return null;
  }
};

const handleFinalSubmit = async () => {
  setIsSubmitting(true);
  const s = useOnboardingStore.getState();

  try {

          const result = await uploadPdf(
            project.pdfLocalUri, 'projects', userId,
            `project-${index}-pdf`,
          );
          pdfUrl = result.url;
        }

        return { ...project, imageUrl, pdfUrl };
      })
    );

    // ── 2. Update profile ──
    await supabase.from('profiles').upsert({
      user_id: userId,
      full_name: s.fullName,
      tagline: s.roles.join(' · '),
      bio: s.bio,
      location: s.location,
      avatar_url: s.avatarUrl,
      resume_url: s.resumeUrl,
      theme: s.theme,
      accent_color: s.accentColor,
      social_github: s.socialGithub,
      social_linkedin: s.socialLinkedin,
      social_twitter: s.socialTwitter,
      social_website: s.socialWebsite,
    }, { onConflict: 'user_id' });

    // ── 3. Insert education ──
    if (s.university) {
      await supabase.from('education').upsert({
        user_id: userId,
        institution: s.university,
        degree: s.degree,
        field: s.field,
        end_year: s.gradYear ? parseInt(s.gradYear) : null,
        is_current: s.gradYear ? parseInt(s.gradYear) >= new Date().getFullYear() : false,
      }, { onConflict: 'user_id' });
    }

    // ── 4. Insert experiences ──
    if (s.experiences.length > 0 && !s.isFresher) {
      // Delete existing, re-insert fresh
      await supabase.from('experiences').delete().eq('user_id', userId);
      await supabase.from('experiences').insert(
        s.experiences.map((exp, i) => ({
          user_id: userId,
          company: exp.company,
          role: exp.role,
          start_date: exp.startDate || null,
          end_date: exp.isCurrent ? null : exp.endDate,
          description: exp.description,
          is_current: exp.isCurrent,
          display_order: i,
        }))
      );
    }

    // ── 5. Insert projects ──
    if (uploadedProjects.length > 0) {
      await supabase.from('projects').delete().eq('user_id', userId);
      await supabase.from('projects').insert(
        uploadedProjects.map((proj, i) => ({
          user_id: userId,
          title: proj.title,
          description: proj.description,
          tech_stack: proj.techStack,
          live_url: proj.liveUrl || null,
          github_url: proj.githubUrl || null,
          image_url: proj.imageUrl,
          pdf_url: proj.pdfUrl,
          featured: i === 0,
          display_order: i,
        }))
      );
    }

    // ── 6. Insert skills ──
    if (s.skills.length > 0) {
      await supabase.from('skills').delete().eq('user_id', userId);
      await supabase.from('skills').insert(
        s.skills.map(skill => ({
          user_id: userId,
          name: skill.name,
          category: skill.category,
        }))
      );
    }

      .from('users')

    // Navigate to generating screen
    router.replace({
      pathname: '/generating',
      params: { buildId },
    });

  } catch (err) {
    setIsSubmitting(false);
    setSubmitError('Something went wrong. Your answers are saved — tap to try again.');
    console.error('Final submit error:', err);
  }
};

STEP 9 — PORTFOLIO BUILD PIPELINE
9A — Portfolio Builder Service (src/services/portfolioBuilder.ts)
typescriptimport { supabase } from '../lib/supabase';
import { useBuildStore } from '../store/buildStore';

const N8N_WEBHOOK_URL = process.env.N8N_WEBHOOK_URL!;
const N8N_SECRET = process.env.N8N_WEBHOOK_SECRET!;

/**
 * Creates a build record in Supabase, then fires the n8n webhook.
 * Returns the build ID so the generating screen can subscribe to it.
 */
export const triggerPortfolioBuild = async (
  userId: string,
  triggeredBy: 'onboarding' | 'update' | 'manual' = 'update',
): Promise<string> => {
  // 1. Create a build record in 'queued' status
  const { data: build, error } = await supabase
    .from('site_builds')
    .insert({
      user_id: userId,
      status: 'queued',
      triggered_by: triggeredBy,
    })
    .select('id')
    .single();

  if (error || !build) throw new Error(`Failed to create build record: ${error?.message}`);

  const buildId = build.id;

  // 2. Fire the n8n webhook (fire-and-forget — n8n handles the rest)
  fetch(N8N_WEBHOOK_URL, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-BEXO-Secret': N8N_SECRET,
    },
    body: JSON.stringify({
      user_id: userId,
      build_id: buildId,
      triggered_by: triggeredBy,
      timestamp: new Date().toISOString(),
    }),
  }).catch(err => {
    // Non-blocking: log but don't throw. The build record exists, n8n will poll.
    console.error('n8n webhook fire failed:', err);
  });

  return buildId;
};
9B — Realtime Build Status Hook (src/hooks/useBuildStatus.ts)
This hook subscribes to the site_builds row in real-time via Supabase Realtime.
The Generating screen uses this to animate progress and know when to navigate to Success.
typescriptimport { useEffect } from 'react';
import { supabase } from '../lib/supabase';
import { useBuildStore } from '../store/buildStore';

export const useBuildStatus = (buildId: string | null) => {
  const { setStatus, setSiteUrl, setRating, setLighthouse, setError } = useBuildStore();

  useEffect(() => {
    if (!buildId) return;

    // Subscribe to realtime changes on this specific build row
    const channel = supabase
      .channel(`build-${buildId}`)
      .on(
        'postgres_changes',
        {
          event: 'UPDATE',
          schema: 'public',
          table: 'site_builds',
          filter: `id=eq.${buildId}`,
        },
        (payload) => {
          const build = payload.new as {
            status: string;
            site_url: string | null;
            rating: number | null;
            lighthouse_perf: number | null;
            lighthouse_a11y: number | null;
            lighthouse_seo: number | null;
            error_log: string | null;
          };

          setStatus(build.status as any);

          if (build.site_url) setSiteUrl(build.site_url);
          if (build.rating) setRating(build.rating);

          if (build.lighthouse_perf && build.lighthouse_a11y && build.lighthouse_seo) {
            setLighthouse(build.lighthouse_perf, build.lighthouse_a11y, build.lighthouse_seo);
          }

          if (build.error_log) setError(build.error_log);
        }
      )
      .subscribe();

    return () => {
      supabase.removeChannel(channel);
    };
  }, [buildId]);
};

IMPORTANT: SECURITY RULES

Never put API keys (OpenAI, Anthropic) in the React Native bundle. They go in Supabase Edge Function environment secrets only. The mobile app calls the edge function, never the AI APIs directly.
The n8n webhook URL and secret can be in the app bundle (it's a one-way trigger with a secret header), but use EAS Secrets for production builds.
Supabase anon key is safe to be public — it's designed to be. RLS policies on every table enforce security.
Resume files are stored in a private bucket. Only the owner can read them. They are never publicly accessible.
User handles are validated server-side by the Postgres CHECK constraint, not just client-side.
All user-generated content (project descriptions, bios) must be passed through the n8n/AI pipeline for moderation before being published to the live portfolio site. Add a content moderation step in n8n using OpenAI Moderation API (free endpoint).
